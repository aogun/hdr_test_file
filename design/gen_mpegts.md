# gen_mpegts.sh - MPEGTS 测试视频生成脚本

## 概述

Bash 脚本，用于生成包含 64 色块测试图案的 MPEGTS 格式视频文件。通过内嵌 Python 代码生成 raw YUV 帧数据，再调用 FFmpeg 编码为 H.264/H.265 MPEGTS 流。

## 依赖

- Python 3（用于生成 YUV 原始帧）
- FFmpeg（libx264 / libx265 编码器）

## 命令行参数

| 参数 | 短选项 | 长选项 | 默认值 | 说明 |
|------|--------|--------|--------|------|
| 帧率 | `-f` | `--fps` | 30 | 输出视频帧率 |
| 编码 | `-c` | `--codec` | h264 | h264 或 h265 |
| 分辨率 | `-r` | `--resolution` | 1080p | 4k / 1080p / 720p |
| 时长 | `-d` | `--duration` | 10 | 视频时长（秒） |
| 采样格式 | `-s` | `--sampling` | yuv420 | yuv444 / yuv422 / yuv420 |
| 输出文件 | `-o` | `--output` | 自动生成 | 输出文件路径 |

默认输出文件名格式：`test_{分辨率}_{编码}_{帧率}fps_{时长}s_{采样}.mpegts`

|------|------|------|

## 测试图案设计

### 64 色块网格

- 8x8 网格，共 64 个色块
- 每个色块的 YUV 值按序号分配：
  - **Y**: 16 ~ 235 均匀分布（ITU-R BT.601 范围）
  - **U**: 128 + sin(idx * pi/32) * 40
  - **V**: 128 + cos(idx * pi/32) * 40

### 文字标注

每个色块绘制两行文字，使用与背景亮度对比的文字颜色（背景 Y > 128 时用 Y=16，否则用 Y=235）：
- 上半区域居中：色块序号（0~63）
- 下半区域居中：完整 YUV 值（格式 `Y,U,V`）

使用 5x5 像素位图字体渲染数字和逗号，每行用低 5 位表示像素。字体大小 = max(min(BLOCK_W, BLOCK_H) // 8, 5)，YUV 子文字缩放为主字号的一半（最小为 1）。

## 编码参数

| 参数 | 值 | 说明 |
|------|----|------|
| `-g` | 等于 FPS | GOP 大小 = 1秒 |
| `-bf` | 0 | 无 B 帧 |
| `-qp` | 0 | 无损量化 |
| `-preset` | ultrafast | 最快编码速度 |
| `-f` | mpegts | 输出容器格式 |

## 处理流程

```
参数解析/验证
    ↓
Python 生成 raw YUV 帧 → 临时 YUV 文件
    ↓
FFmpeg 编码 (rawvideo → H.264/H.265 MPEGTS)
    ↓
清理临时文件，输出结果
```

## YUV 帧生成

Python 内嵌脚本负责生成原始 YUV 帧数据：

1. 以 YUV444 方式生成完整的 Y/U/V 三个平面
2. 根据采样格式对 U/V 色度平面进行子采样：
   - yuv444：不做子采样，U/V 平面与 Y 平面相同尺寸
   - yuv422：水平 2:1 子采样，U/V 平面宽度减半
   - yuv420：水平 + 垂直 2:1 子采样，U/V 平面宽高均减半
3. 逐帧写入临时 YUV 文件（planar 格式：先全部 Y，再全部 U，再全部 V）
4. 所有帧内容相同（静态图案），不含帧间变化
